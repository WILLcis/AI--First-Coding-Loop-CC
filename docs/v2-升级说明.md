# Harness Kit v2 升级说明

> v2 把 Addy Osmani《Loop Engineering》的 5 块积木全部内化,并加上反认知投降的护栏。
> **核心:harness(运行环境)+ loop(在 harness 里跑的小回路)= 完整 AI-First 系统。**

## 7 个升级一览(按收益 × 成本排序)

| # | 升级 | 新增/改动 | 收益 |
|---|---|---|---|
| 1 | **Skills 系统** | `skills/<name>/SKILL.md` × 6 + `skills/README.md` | agent 上下文按需加载;跨工具可移植到 `.claude/skills/` 或 `.codex/skills/` |
| 2 | **Sub-agent 模型分层** | `agents/*.toml` × 7;改 `claude-review.yml` 三趟用三种模型 | API 月费降 **40~60%**;security 趟反而升级到 Opus+high reasoning |
| 3 | **/goal Loop 范式** | `scripts/goal_loop.py` | maker/checker 显式分离;跑到验证条件成立才停,而非定时跑一次 |
| 4 | **Worktree 隔离** | `scripts/spawn_agent_worktree.sh` | 并行 agent 任务不撞 fs / docker / lock |
| 5 | **state/ 显式记忆** | `state/{triage-history,token-usage,comprehension-log}.jsonl` + `tasks/` + `known-flakes.txt` | agent 不再失忆;回归识别更准;一切可审计 |
| 6 | **Token 预算追踪** | `scripts/_adapters.record_token_usage` + `scripts/token_report.py` | 账单可观察、可预算、超 80% 告警 |
| 7 | **反认知投降护栏** | `skills/weekly-comprehension-check/` + `scripts/comprehension_metrics.py` + `skills/README.md` | 三项硬指标低于红线触发警报;不让 harness 把架构师变成按钮工 |

## 概念映射(Addy 5 块积木 → 我们的实现)

| Addy 的"积木" | v1 | v2 |
|---|---|---|
| Automations | daily-health + triage cron | + goal_loop("跑到条件成立") |
| Worktrees | 仅 git 分支 | **`spawn_agent_worktree.sh`** + COMPOSE_PROJECT_NAME 隔离 |
| Skills | 单 CLAUDE.md + prompts/ | **`skills/`** 体系,YAML front-matter 触发 |
| Plugins/Connectors | `scripts/_adapters.py` | 同(已经够好) |
| Sub-agents | 三趟同质评审 | **`agents/`** 7 个角色 TOML + 模型分层 |
| State/Memory | Linear + git 隐式 | **`state/`** 目录,append-only 显式 |

## 升级到 v2 的迁移步骤(若你已部署 v1)

> 全部向后兼容:v1 的 prompts/ 仍在工作,v2 只是叠加更多能力。

1. **拉取 v2 文件**:把 `skills/`、`agents/`、`state/`、`scripts/{goal_loop,spawn_agent_worktree,token_report,comprehension_metrics}` 合到现有仓库
2. **更新 workflow**:`claude-review.yml`、`daily-health.yml` 改为 v2 版本(差异主要是模型字段与新的 env)
3. **配新的 secret/vars**:
   - `GH_API_TOKEN`(读 PR/review 的 PAT,给 comprehension_metrics 用)
   - vars `ARCHITECT_LOGIN`(你的 GitHub 用户名)
   - vars `MONTHLY_TOKEN_BUDGET`(粗估月 token 上限,可留空)
4. **首次跑**:本地 `python3 scripts/comprehension_metrics.py --mock`、`python3 scripts/token_report.py` 确认输出形态
5. **写第一个 task**:用 `skills/architect-task-writer/SKILL.md` 的格式写一份,存到 `state/tasks/<id>.md`,跑 `scripts/goal_loop.py --task-id <id> --task-file ... --stop "..."`
6. **每周一**:做 `skills/weekly-comprehension-check/SKILL.md` 的 15 分钟自检,写入 `state/comprehension-log.jsonl`

## 哪些东西 v2 *没动*(避免升级焦虑)

- `Makefile`、`docker-compose.*`、`flags/feature-flags.ts`、`prompts/*.md` 全部保留
- `ci.yml`、`deploy.yml`、`triage.yml`(workflow 入口)的整体形态不变
- `verify_triage.py`、`gen_release_notes.py`、`check_env_parity.py` 不动
- 整个迁移可在一周内完成,无需停服或重构业务代码
